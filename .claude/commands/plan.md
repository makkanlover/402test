# 計画モード起動

調停者として、開発タスクの計画を立案してください。

## 実施内容

1. **要求分析**
   - ユーザーの要求を詳細に分析
   - 必要な機能を洗い出し

2. **アーキテクチャ設計**
   - 全体構造の設計
   - 使用する技術スタックの選定

3. **タスク分割**
   - 具体的なタスクに分割
   - 優先順位の設定
   - 各タスクに適したエージェントの割り当て

4. **リスク分析**
   - 潜在的な問題点の特定
   - トレードオフの明示

## ユーザーの要求
$ARGUMENTS

## 出力形式
「調停者レポート」形式で計画を報告すること。
